## WiFi SSIDs

| SSID | Purpose | Subnet | Notes |
|---|---|---|---|
| ssid-home | Main / trusted devices | 192.168.xx.xx/24 | Active |
| ssid-guest | Guests | 192.168.xx.xx/24 (target) | Active — wireless isolation only, no VLAN |
| ssid-iot | IoT devices | 192.168.xx.xx/24 (target) | Active — wireless isolation only, no VLAN |

No VLAN tagging in place — all SSIDs share the flat `192.168.xx.xx/24` network. Isolation is wireless-level only. See [iot.md](iot.md) for device inventory and [security.md](security.md) for the threat model.


## Remote access

| Component | Detail |
|---|---|
| VPN | Tailscale — hosted on clementi |
| Tailscale features | Exit node, Funnel (public HTTPS), subnet routing |
| SSL certificates | Let's Encrypt via Cloudflare DNS challenge — issued per service via nginx-proxy-manager |
| Public webhook cert | Tailscale-issued certificate for n8n Funnel endpoint |
| Public webhook | Tailscale Funnel → braddell:5678 (n8n) |
| Local DNS | nginx-proxy-manager only — no Pi-hole or local resolver |

See [services.md](services.md) for the full service map including Tailscale and nginx-proxy-manager. See [security.md](security.md) for the attack surface and exposed endpoints.
