# Scripts and Workflows

[← README](../README.md)

Reference for all scripts in `_scripts/` and CI workflows in `.gitea/workflows/`.

Scripts follow a verb-prefix naming convention: `setup-*` (one-time init), `generate-*` (derive snapshot artifacts), `check-*` (read-only status), `update-*` (apply changes to running systems).

---

## CI Workflows

| Workflow | Trigger | What it does |
|---|---|---|
| `snapshots.yaml` | Weekly (Mon 06:00 UTC) + manual | Gates on backup status, then runs all `generate-*` snapshot scripts and commits results to `_snapshots/` |
| `generate-setup-scripts.yaml` | Manual only | SSHes to changi as `homelab-admin`, runs `generate-host-setup.sh` + `generate-vm-setup.sh`, retrieves output into `changi/_setup/` and `_snapshots/`, commits |
| `update-docs.yaml` | Push to `_snapshots/**` + manual | Runs `update-docs-agent.py` — sends snapshots to Ollama (gemma4 on braddell), applies proposed doc edits, opens a PR for review |
For secrets required by each workflow, see [secrets.md](secrets.md) — Gitea Actions secrets table.

## Snapshots reference

All snapshot files land in `_snapshots/` and are committed by CI weekly (snapshots workflow). See [backup.md](backup.md) for context.

| File | Generated by | Contents |
|---|---|---|
| `image-hash-{host}.yaml` | `generate-image-snapshot.sh` | Current image SHA per container — pre-update rollback reference |
| `network-hosts.yaml` | `generate-network-snapshot.sh` | Full host→IP map: DHCP reservations (with MACs) from florence + static-IP hosts (Proxmox VMs, Pi, NAS) |
| `npm.yaml` | `generate-npm-snapshot.sh` | nginx-proxy-manager proxy host table |
| `pve-config.yaml` | `generate-pve-config-snapshot.sh` | Proxmox storage pools, backup jobs, network interfaces, VM summary |
| `pbs-config.yaml` | `generate-pbs-config-snapshot.sh` | PBS datastore status/usage, prune jobs, verify jobs |
| `kopia-config.yaml` | `generate-kopia-config-snapshot.sh` | Combined kopia config across all VMs — global policy, per-host path policies (full), repository connection, latest snapshot per source |
| `vm-{vmid}-{name}.yaml` | `generate-vm-setup.sh` | Full per-VM Proxmox config — hardware, disks, network, PCIe passthrough, options |

## generate-*

| Script | Runs on | What it does | Doc |
|---|---|---|---|
| `generate-network-snapshot.sh` | CI runner (bedok) | SSHes to florence for DHCP reservations (with MACs) + SSH probes static-IP hosts → `_snapshots/network-hosts.yaml`. Complete host→IP+MAC reference. | [network.md](network.md) |
| `generate-npm-snapshot.sh` | CI runner (bedok) | Calls NPM API at `proxy.home.lab`, dumps proxy hosts → `_snapshots/npm.yaml` | [services.md](services.md) |
| `generate-image-snapshot.sh` | Target VM | Outputs current image SHA for every running Docker container on that host → stdout. Run by CI weekly; also used manually for a pre-update rollback reference. | [provisioning.md](provisioning.md) |
| `generate-vm-setup.sh` | changi | Reads current Proxmox VM state → `changi/_setup/{vmid}-{name}.sh` (creation scripts) and `_snapshots/vm-{vmid}-{name}.yaml` (full config snapshots). Run after any VM hardware change. | [provisioning.md](provisioning.md) |
| `generate-host-setup.sh` | changi | Reads current Proxmox host config (storage pools, backup jobs, network interfaces) → `changi/_setup/changi-setup.sh`. Run after any host config change. | [provisioning.md](provisioning.md) |
| `generate-pve-config-snapshot.sh` | CI runner (bedok) | Calls Proxmox API on changi — snapshots storage pools, backup jobs, network interfaces, VM summary → `_snapshots/pve-config.yaml`. Requires `PROXMOX_TOKEN_ID` + `PROXMOX_TOKEN_SECRET`. | [backup.md](backup.md) |
| `generate-pbs-config-snapshot.sh` | CI runner (bedok) | Calls PBS API on verona — snapshots datastore status, prune jobs, verify jobs → `_snapshots/pbs-config.yaml`. Requires `PBS_TOKEN_ID` + `PBS_TOKEN_SECRET`. | [backup.md](backup.md) |
| `generate-kopia-config-snapshot.sh` | CI runner (bedok) | SSHes to each VM, runs `docker exec kopia` commands — snapshots full kopia config (global policy, per-path policies, repo connection, latest snapshots) into one combined `_snapshots/kopia-config.yaml`. | [backup.md](backup.md) |

## check-*

| Script | Runs on | What it does | Doc |
|---|---|---|---|
| `check-backup-status.sh` | changi | Verifies PBS last backup age and kopia last snapshot age across all VMs. Exit 0 = all good, exit 1 = one or more checks failed. Required gate before starting any update run. | [update.md](update.md) |

## update-*

| Script | Runs on | What it does | Doc |
|---|---|---|---|
| `update-containers.sh` | Target VM | Pulls latest images and redeploys all stacks for that host. Run after `check-backup-status` passes and image hashes are snapshotted. Mirrors `deploy.sh` but pulls before each service. | [update.md](update.md) |
| `update-docs-agent.py` | bedok (via CI) | Reads `_snapshots/` and current `_docs/`, sends to Ollama (gemma4 on braddell), applies JSON find/replace edits proposed by the model, then opens a PR on `docs/auto-update-{date}` for review. Triggered by CI on push to `_snapshots/**`. | — |
